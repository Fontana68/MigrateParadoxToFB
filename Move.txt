## 1. Choose Your Local Database

Since you want data saved locally on the PC, you have three excellent FireDAC-native options:

| Database | Best For | File Type | Deployment |
|----------|----------|-----------|------------|
| **SQLite** | Simple local storage, single-file, zero-config | `.db`, `.sqlite` | Single DLL (`sqlite3.dll`) or static linking |
| **InterBase (IBLite/IBToGo)** | Embarcadero-optimized, strong encryption, easy migration from BDE Paradox | `.ib` | Run-time royalty-free with RAD Studio license |
| **Firebird Embedded** | Full SQL power, scales to client/server later | `.fdb` | Single DLL (`fbclient.dll`) |

**Recommendation:** Use **SQLite** if you want the simplest, most widely supported single-file local database. Use **InterBase** if you want tight RAD Studio integration and easy migration from Paradox/dBase formats.

## 2. Automated Code Migration with `reFind`

RAD Studio includes the **`reFind.exe`** tool that performs regex-based search-and-replace to convert BDE components to FireDAC. In RAD Studio 12.3, Embarcadero continues to maintain and improve these migration scripts. 

### Location
```
C:\Program Files (x86)\Embarcadero\Studio\23.0\bin\reFind.exe
```
Migration scripts are in the demos folder:
```
C:\Users\Public\Documents\Embarcadero\Studio\23.0\Samples\Object Pascal\Database\FireDAC\Tool\reFind\
```

### How to Run
1. **Back up your project** completely.
2. Open a command prompt in your project directory.
3. Run the BDE-to-FireDAC script:
   ```cmd
   "C:\Program Files (x86)\Embarcadero\Studio\23.0\bin\reFind.exe" *.pas *.dfm /I /W /P:TQuery /R:TFDQuery
   ```
   Or use the provided script files (e.g., `BDE2FDMigration.txt`) for comprehensive replacement. 

### What `reFind` Converts Automatically
| BDE Component | FireDAC Equivalent |
|---------------|-------------------|
| `TDatabase` | `TFDConnection` |
| `TQuery` | `TFDQuery` |
| `TTable` | `TFDTable` |
| `TSession` | `TFDManager` (usually not needed) |
| `TBatchMove` | `TFDBatchMove` |
| `TUpdateSQL` | `TFDUpdateSQL` (often unnecessary) |

## 3. Manual Steps After `reFind`

### A. Update Your Uses Clauses
Remove:
```pascal
uses
  DBTables, BDE;
```
Add:
```pascal
uses
  FireDAC.Stan.Intf, FireDAC.Stan.Option, FireDAC.Stan.Error,
  FireDAC.UI.Intf, FireDAC.Phys.Intf, FireDAC.Stan.Def,
  FireDAC.Stan.Pool, FireDAC.Stan.Async, FireDAC.Phys,
  FireDAC.Phys.SQLite, FireDAC.VCLUI.Wait,  // or FireDAC.Phys.IB, FireDAC.Phys.FB
  FireDAC.Comp.Client, FireDAC.Comp.DataSet;
```

### B. Replace Database Connection Setup

**Old BDE (`TDatabase`):**
```pascal
Database1.AliasName := 'DEFAULTDD';
Database1.DatabaseName := 'MyDB';
Database1.Connected := True;
```

**New FireDAC (SQLite local file):**
```pascal
FDConnection1.Params.DriverID := 'SQLite';
FDConnection1.Params.Database := ExtractFilePath(ParamStr(0)) + 'Data\mydata.db';
FDConnection1.Params.Add('OpenMode=CreateUTF8'); // creates if missing
FDConnection1.Connected := True;
```

**New FireDAC (InterBase local):**
```pascal
FDConnection1.Params.DriverID := 'IB';
FDConnection1.Params.Database := ExtractFilePath(ParamStr(0)) + 'Data\mydata.ib';
FDConnection1.Params.UserName := 'sysdba';
FDConnection1.Params.Password := 'masterkey';
FDConnection1.Params.Add('CreateDatabase=Yes');
FDConnection1.Connected := True;
```

### C. Update Query/Table Properties

| BDE Property | FireDAC Equivalent |
|-------------|-------------------|
| `DatabaseName` | `Connection` (points to TFDConnection) |
| `SQL` | `SQL` (same, but enhanced syntax) |
| `RequestLive` | Not needed; use `UpdateOptions.UpdateTableName` |
| `ParamByName` | `ParamByName` (same API) |
| `FieldByName` | `FieldByName` (same API) |

## 4. Migrate the Actual Data

You need to move data from BDE formats (Paradox `.db`, dBase `.dbf`) into your new local database.

### Option A: Using `TFDBatchMove` (Recommended)
FireDAC's `TFDBatchMove` component can transfer data from BDE datasets to FireDAC datasets.

```pascal
uses FireDAC.Comp.BatchMove, FireDAC.Comp.BatchMove.DataSet;

procedure MigrateBDEToSQLite;
var
  BatchMove: TFDBatchMove;
  Reader: TFDBatchMoveDataSetReader;
  Writer: TFDBatchMoveDataSetWriter;
  BDEQuery: TQuery;  // your old BDE query
begin
  BDEQuery := TQuery.Create(nil);
  try
    BDEQuery.DatabaseName := 'MyBDEAlias';
    BDEQuery.SQL.Text := 'SELECT * FROM Customers';
    BDEQuery.Open;

    FDQuery1.SQL.Text := 'SELECT * FROM Customers WHERE 1=0';
    FDQuery1.Open;

    BatchMove := TFDBatchMove.Create(nil);
    Reader := TFDBatchMoveDataSetReader.Create(BatchMove);
    Writer := TFDBatchMoveDataSetWriter.Create(BatchMove);
    try
      Reader.Dataset := BDEQuery;
      Writer.Dataset := FDQuery1;
      BatchMove.Execute;
    finally
      BatchMove.Free;
    end;
  finally
    BDEQuery.Free;
  end;
end;
```

### Option B: Export/Import Tools
- Use **InterBase Data Pump** or third-party tools to move Paradox/dBase to SQLite or InterBase.
- For SQLite, tools like **DB Browser for SQLite** can import CSV exports from BDE tables.

## 5. RAD Studio 12.3 Specific Setup

### Form Designer Changes
1. Drop a **`TFDConnection`** on your DataModule/Main Form.
2. Drop a **`TFDPhysSQLiteDriverLink`** (or `TFDPhysIBDriverLink`) on the form — this ensures the driver is linked into your executable.
3. Set `TFDQuery.Connection` = `FDConnection1`.

### Connection at Runtime (Relative Path)
Always use a path relative to your EXE for true local deployment: 
```pascal
procedure TDataModule1.DataModuleCreate(Sender: TObject);
var
  dbPath: string;
begin
  dbPath := ExtractFilePath(ParamStr(0)) + 'Database\appdata.db';
  ForceDirectories(ExtractFilePath(dbPath));

  FDConnection1.Params.Values['Database'] := dbPath;
  FDConnection1.Connected := True;
end;
```

### SQLite-Specific Notes
- SQLite is included in all RAD Studio editions (including Community Edition) with no deployment restrictions for local use. 
- For a single-file database with no external DLL, you can use the **statically linked SQLite** option in FireDAC (set `DriverID=SQLite` and ensure `SQLiteLibrary` parameter is empty for static linking in some configurations, though typically `sqlite3.dll` is required).

## 6. Common Pitfalls & Fixes

| Issue | Solution |
|-------|----------|
| **"Table is read-only"** | Set `TFDQuery.UpdateOptions.UpdateTableName` and ensure `TFDQuery.UpdateOptions.KeyFields` is set. |
| **Date/Time format errors** | SQLite stores dates as text or integers; use `FormatSettings` or FireDAC's `DataTypeMap` to handle BDE date formats. |
| **AutoInc fields** | In SQLite, define integer primary key as `INTEGER PRIMARY KEY AUTOINCREMENT`. In InterBase, use `GENERATOR`. |
| **Locate/Lookup differences** | FireDAC's `Locate` works similarly, but case-sensitivity may differ from BDE. Use `loCaseInsensitive`. |
| **BDE-specific methods** | Replace `Session.GetAliasNames`, `Session.GetTableNames` with `TFDManager.GetConnectionNames` or `TFDConnection.GetTableNames`. |

## 7. Quick Checklist

1. **Backup** your entire project and BDE data.
2. Run **`reFind`** with the BDE-to-FireDAC script.
3. Fix `uses` clauses and remove BDE references.
4. Add `TFDConnection` and configure for SQLite/InterBase.
5. Replace `TDatabase` references with `TFDConnection`.
6. Migrate data using `TFDBatchMove` or external tools.
7. Test all `Locate`, `Filter`, and `Edit/Post` operations.
8. Deploy with the appropriate driver DLL (if needed) and local database file.

## 1. File necessari per Firebird Embedded

A partire da Firebird 3.0/4.0, la modalitŕ embedded non usa piů `fbembed.dll`, ma direttamente **`fbclient.dll`** con una configurazione specifica .

Crea una cartella (es. `.\Firebird_Embedded\`) accanto al tuo EXE e copia questi file minimi (Win32):

| File | Note |
|------|------|
| `fbclient.dll` | Client library principale |
| `firebird.conf` | Configurazione; imposta `Providers = Engine13` |
| `firebird.msg` | Messaggi di errore |
| `ib_util.dll` | Utility interna |
| `icudt*.dll`, `icuin*.dll`, `icuuc*.dll` | ICU libraries (versione varia, es. 63, 70...) |

| `plugins\engine13.dll` | Motore database (per FB 3.x/4.x) |
| `msvcp140.dll`, `vcruntime140.dll` | VC++ Redistributable |

> **Nota bitness**: se compili **Win32**, devi usare la versione a 32 bit di tutti i file DLL .

## 2. Configurazione FireDAC per Embedded

Sul DataModule o Form principale:

```pascal
uses
  FireDAC.Stan.Intf, FireDAC.Stan.Option, FireDAC.Comp.Client,
  FireDAC.Phys.FB, FireDAC.Phys.FBDef, FireDAC.VCLUI.Wait;

procedure TDataModule1.SetupFirebirdEmbedded;
var
  dbPath: string;
begin
  dbPath := ExtractFilePath(ParamStr(0)) + 'MOT.fdb';

  // --- Driver Link ---
  FDPhysFBDriverLink1.VendorHome := ExtractFilePath(ParamStr(0)) + 'Firebird_Embedded';
  FDPhysFBDriverLink1.VendorLib  := 'fbclient.dll';
  FDPhysFBDriverLink1.Embedded   := True;  // Modalitŕ embedded!

  // --- Connessione ---
  FDConnection1.DriverName := 'FB';
  FDConnection1.Params.Values['Protocol'] := 'Local';
  FDConnection1.Params.Values['Server']   := '';        // VUOTO per embedded
  FDConnection1.Params.Values['Database'] := dbPath;
  FDConnection1.Params.Values['User_Name']:= 'SYSDBA';
  FDConnection1.Params.Values['Password'] := 'masterkey';
  FDConnection1.Params.Values['CharacterSet'] := 'UTF8';

  FDConnection1.LoginPrompt := False;
end;
```

**Punti critici** :
- `Server` deve essere **stringa vuota**
- `Protocol` deve essere **`Local`**
- `Embedded := True` sul `TFDPhysFBDriverLink`

## 4. Migrare lo SCHEMA (struttura tabelle)

FireDAC non converte automaticamente lo schema Paradox ? Firebird. Hai tre opzioni:

### Opzione A: Tool esterno (consigliata per tante tabelle)
Usa **IBDataPump** (freeware di CleverComponents) che supporta direttamente Paradox ? Firebird , oppure strumenti commerciali come Full Convert .

### Opzione B: Script SQL manuale
Ricrea le tabelle in Firebird usando i tipi dati equivalenti:

| Paradox | Firebird |
|---------|----------|
| `String[n]` / `Alpha` | `VARCHAR(n)` |
| `Short` | `SMALLINT` |
| `LongInteger` | `INTEGER` |
| `AutoIncrement` | `INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY` |
| `Number` (float) | `DOUBLE PRECISION` |
| `Currency` / `Money` | `DECIMAL(18,4)` |
| `Date` | `DATE` |
| `Time` | `TIME` |
| `Timestamp` | `TIMESTAMP` |
| `Memo` | `BLOB SUB_TYPE TEXT` |
| `Graphic` / `Blob` | `BLOB SUB_TYPE BINARY` |
| `Logical` | `BOOLEAN` (FB 3.0+) o `SMALLINT` |

Esempio creazione tabella:
```sql
CREATE TABLE CLIENTI (
  ID INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  NOME VARCHAR(100) NOT NULL,
  DATA_NASCITA DATE,
  SALDO DECIMAL(18,2),
  NOTE BLOB SUB_TYPE TEXT,
  ATTIVO BOOLEAN DEFAULT TRUE
);
```

### Opzione C: Generazione automatica via codice
Puoi scrivere un programma che legge `TTable.FieldDefs` (BDE) e genera gli `ALTER TABLE`/`CREATE TABLE` per Firebird.

## 5. Migrare i DATI con TFDBatchMove

Una volta create le tabelle vuote in Firebird, usa `TFDBatchMove` per trasferire i dati :

```pascal
uses
  FireDAC.Comp.BatchMove, FireDAC.Comp.BatchMove.DataSet;

procedure TDataModule1.MigrateTable(const BDETableName, FBTableName: string);
var
  BDETable: TTable;           // componente BDE
  FBQuery: TFDQuery;          // componente FireDAC
  BatchMove: TFDBatchMove;
  Reader: TFDBatchMoveDataSetReader;
  Writer: TFDBatchMoveDataSetWriter;
begin
  // --- Sorgente BDE (Paradox) ---
  BDETable := TTable.Create(nil);
  try
    BDETable.DatabaseName := 'NOME_ALIAS_BDE';  // il tuo alias BDE
    BDETable.TableName    := BDETableName;      // es. 'CLIENTI.DB'
    BDETable.Open;

    // --- Destinazione Firebird ---
    FBQuery := TFDQuery.Create(nil);
    try
      FBQuery.Connection := FDConnection1;
      FBQuery.SQL.Text := 'SELECT * FROM ' + FBTableName + ' WHERE 1=0';
      FBQuery.Open;  // dataset vuoto ma con fielddefs

      // --- BatchMove ---
      BatchMove := TFDBatchMove.Create(nil);
      try
        Reader := TFDBatchMoveDataSetReader.Create(BatchMove);
        Writer := TFDBatchMoveDataSetWriter.Create(BatchMove);

        Reader.DataSet := BDETable;
        Writer.DataSet := FBQuery;

        // Opzionale: mappatura campi se i nomi differiscono
        // BatchMove.Mappings.Add('NOME_PARADOX=NOME_FIREBIRD');

        BatchMove.Execute;
        ShowMessage(Format('Migrati %d record in %s', [BatchMove.MovedCount, FBTableName]));
      finally
        BatchMove.Free;
      end;
    finally
      FBQuery.Free;
    end;
  finally
    BDETable.Free;
  end;
end;
```

**Uso**:
```pascal
FDConnection1.Connected := True;
MigrateTable('CLIENTI.DB', 'CLIENTI');
MigrateTable('ORDINI.DB',  'ORDINI');
```

## 6. Considerazioni importanti

| Aspetto | Soluzione |
|---------|-----------|
| **Percorso database** | Usa sempre percorsi assoluti o relativi all'EXE. Non usare alias BDE per il file FDB. |
| **Charset** | Paradox usa spesso `WIN1252`; Firebird Embedded usa meglio `UTF8`. Verifica le stringhe accentate dopo la migrazione. |
| **Indici** | Ricrea gli indici in Firebird con `CREATE INDEX`; Paradox li gestiva diversamente. |
| **Chiavi primarie** | Sostituisci le chiavi Paradox con `PRIMARY KEY` o `GENERATED ALWAYS AS IDENTITY`. |
| **Transazioni** | Firebird č transazionale rigoroso. Se usi `TFDBatchMove`, la scrittura č automaticamente transazionale. |
| **Deployment** | Distribuisci i file della sezione 1 nella stessa cartella dell'EXE (o in una sottocartella referenziata da `VendorHome`). |

## 7. Riepilogo passaggi

1. **Prepara** la cartella con i file Firebird Embedded (32 bit).
2. **Configura** `TFDPhysFBDriverLink` (`Embedded=True`) e `TFDConnection` (`Protocol=Local`, `Server=`).
3. **Crea** `MOT.fdb` con `CREATE DATABASE` se non esiste.
4. **Crea** le tabelle in Firebird (manuale, script o tool esterno come IBDataPump ).
5. **Trasferisci** i dati con `TFDBatchMove` da `TTable` (BDE) a `TFDQuery` (Firebird).
6. **Verifica** charset, date e campi memo/BLOB.
7. **Rilascia** l'applicazione con i DLL di Firebird nella cartella dell'EXE.
